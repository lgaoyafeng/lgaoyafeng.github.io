Dockerfile

dockerfile是用来构建docker镜像的命令参数脚本

构建步骤

  1. 编写dockerfile文件
  2. docker build成为一个镜像
  3. docker run运行镜像
  4. docker pull 上传镜像(DockerHub, 阿里云镜像仓库 )

官方的镜像大部分都是基础镜像,好多功能都没有,我们通常都会构建自己的镜像

Dockerfile 命令

dockerfile构建过程

  1. 每一个保留关键字(指令)都必须是大写字母
  2. 执行顺序从上到下
  3. #表示注释
  4. 每一指令都会提交一个新的镜像层,并提交

dockerfile逐渐成为企业交付的标准, 必须要掌握

步骤: 开发->部署->运维

Dockerfile :构建文件, 定义了一切的步骤,源码

Dockerimages: 通过dockerfile构建的镜像, 最终发布和运行的产品

Docker容器: 容器就是镜像运行起来的服务器
